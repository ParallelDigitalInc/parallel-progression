The AI Native Design Foundations

Designing with AI

From asking a chatbot for help → to running a system that designs alongside you.

L1
Level 1
DabblerYou

One-off asks in a chat window. Single-turn, reactive, starts from zero every session.

Spot it →No saved context or setup. Output quality swings wildly because the model never has the background.
Example →Pastes a screen in, asks “improve the copy,” ships whatever comes back.
L2
Level 2
OperatorYou

Uses AI consistently for recurring tasks and writes constrained prompts. Still hand-drives every step.

Spot it →A few reused go-to prompts, can iterate on tone, but it’s all manual, one task at a time.
Example →Reliably drafts empty states, microcopy and first-pass flows before refining in Figma.
L3
Level 3
FluentYou

Sets the AI up before working: a living context file (brief, audience, voice, system rules). Prototypes with a real model, not static mockups.

Spot it →Keeps a context doc; chains steps; evaluates output instead of accepting it. The honest test: would you feel the loss if AI were removed?
Example →Builds a live-model prototype of a search-and-filter flow, sees responses take ~6s, and designs the latency state around real output.
L4
Level 4
OrchestratorYou

Designs the workflow itself. Delegates multi-step work to agents and builds reusable scaffolding (skills, prompt-specs, MCP), then spends energy on direction and the quality bar.

Spot it →The conductor: hands the AI the right context, components and criteria, then judges the result. Builds assets the team reuses.
Example →Ships a reusable “Parallel screen generator” skill, with design tokens, voice and component rules baked in, that the whole studio runs and curates.

Designing for AI

From bolting on a chatbot → to designing the whole intelligent loop the user lives in.

L1
Level 1
Bolt-onYou

Treats AI as a feature to add (“put a chatbot on it”). Designs only the happy path and the magic demo.

Spot it →The flow assumes AI always works. Onboarding over-promises “magic.” No failure states.
Example →A “Magic Fix” button that returns a flawless result in the mock, with no thought to when it doesn’t.
L2
Level 2
Pattern-awareYou

Matches the interaction pattern to the task (automate vs. augment vs. converse) instead of chatbot-for-everything.

Spot it →Can justify why a feature is a background agent vs. a chat vs. an inline suggestion.
Example →Booking a slot = a silent background agent. A risky judgement call = AI flags anomalies for a human to decide.
L3
Level 3
Trust & failure designerYou

Designs for the messy reality of models: graceful failure, honest confidence, disclosure, latency and variance.

Spot it →Every AI surface has a fallback. Copy communicates limits. The bot is clearly labelled a bot.
Example →“Here’s who we found” + a manual fix instead of an error. Support chat discloses it’s automated and sells the benefit, not the tech.
L4
Level 4
Systems & feedback designerYou

Designs the full loop: feedback signals that teach the system, the right success metrics, and agents as a second kind of “user.”

Spot it →Success is a correction / edit rate, not a vanity count. Explicit and implicit feedback are built into the UI.
Example →An AI draft generator measured by edit distance; a quick dismiss wired as a negative signal that visibly updates suggestions.
